🎯 Módulo 1: Planejamento de Auditoria de Sistemas

O que é o planejamento em auditoria?

É a fase inicial onde se define escopo (o que será auditado) e objetivos (por que e para quê). Um bom planejamento evita desperdício de tempo e foca nos riscos reais.

Definição de Escopo

O escopo delimita os sistemas, processos, áreas geográficas ou unidades organizacionais que serão avaliados. Exemplos comuns:

  • Sistemas críticos (internet banking, ERP, banco de dados de clientes)
  • Processos específicos (gestão de acessos, geração de logs, resposta a incidentes)
  • Período de análise (últimos 6 meses, por exemplo)

Definição de Objetivos

Os objetivos respondem: "O que queremos verificar e por quê?". Eles devem estar alinhados aos objetivos de negócio e aos riscos identificados.

📌 Exemplo de objetivo de auditoria:
"Verificar se os controles de acesso ao sistema de internet banking são eficazes para prevenir acessos não autorizados, conforme a política de segurança da informação e a ISO 27001."

Tríade CID como base dos objetivos

  • Integridade: Os dados não foram alterados sem autorização?
  • Confidencialidade: O acesso é restrito a quem realmente precisa?
  • Disponibilidade: Os sistemas estão acessíveis quando necessários?
💡 Na prática: Ao definir objetivos, sempre vincule a uma ocorrência ou risco. Ex: "Devido à falha de logs, verificaremos se o sistema de auditoria está configurado corretamente."

⚠️ Módulo 2: Avaliação e Priorização de Riscos

Conceitos fundamentais

  • Ameaça: Algo que pode causar dano (ex: hacker, ex-funcionário malicioso).
  • Vulnerabilidade: Falha que permite a ameaça se concretizar (ex: credencial ativa de ex-funcionário, falta de logs).
  • Risco: Probabilidade de uma ameaça explorar uma vulnerabilidade, causando impacto ao negócio.
  • Impacto: Consequência financeira, reputacional, legal ou operacional.

Como priorizar riscos?

Utilize a matriz de risco (impacto x probabilidade). Riscos com alto impacto e alta probabilidade são prioritários.

Probabilidade / ImpactoBaixo ImpactoAlto Impacto
Alta ProbabilidadeMédioCRÍTICO (prioridade 1)
Baixa ProbabilidadeBaixoMédio-Alto

Exemplos de riscos em instituições financeiras

  • Acesso não autorizado a contas de clientes → risco de fraude, perda financeira, dano reputacional.
  • Credenciais ativas de ex-funcionários → risco de vazamento de dados ou ações maliciosas internas.
  • Falha na geração de logs de auditoria → impossibilidade de rastrear ações suspeitas, violação de conformidade (LGPD, ISO 27001).
🎯 Priorização: Em geral, riscos que afetam diretamente dados sensíveis de clientes ou compliance regulatório são os mais críticos.

🔐 Módulo 3: Controles Internos e ISO 27001

O que são controles internos?

São medidas (políticas, processos, ferramentas) que reduzem riscos. Podem ser preventivos, detectivos ou corretivos.

Principais controles para auditoria de sistemas

🔑 Controles de Acesso

  • RBAC (Role-Based Access Control): Acesso baseado em função.
  • Revogação imediata de credenciais: Ao desligar um funcionário, o acesso deve ser removido em até 24h (ideal: imediato).
  • MFA (Autenticação Multifator): Exige mais que senha para acessar sistemas críticos.
  • Revisão periódica de acessos: A cada 3 ou 6 meses.

📝 Controles de Logging e Monitoramento

  • Geração obrigatória de logs: Quem, quando, o que fez, de onde.
  • Proteção contra alteração de logs: Logs imutáveis (WORM - Write Once Read Many).
  • Centralização de logs (SIEM): Ferramentas como Splunk, ELK.
  • Alertas automáticos: Para atividades suspeitas (ex: múltiplas tentativas de acesso falhas).

🔍 Módulo 4: Coleta de Evidências em Auditoria

Tipos de métodos de coleta

  • 📄 Revisão documental: Políticas, procedimentos, contratos, relatórios anteriores.
  • 💬 Entrevistas: Com gestores, administradores de sistema, equipe de segurança.
  • 📊 Análise de logs: Verificar se logs existem, estão completos e protegidos.
  • 🧪 Testes técnicos: Simulações, varreduras, scripts de validação.
  • 👀 Observação direta: Acompanhar processos em tempo real.

Exemplos práticos de evidências

Risco / PerguntaEvidência adequada
Ex-funcionário ainda tem acesso?Lista de usuários ativos x lista de desligados (entrevista RH + consulta técnica)
Logs estão sendo gerados?Extração de registros de log de 72h + verificação de falhas de configuração
Acessos não autorizados?Análise de logs de acesso com SQL/Python, buscando padrões anômalos
# Exemplo: consulta SQL para identificar acessos suspeitos em horário noturno SELECT usuario, data_hora, acao, ip_origem FROM logs_acesso WHERE data_hora BETWEEN '2025-01-01 00:00:00' AND '2025-01-01 06:00:00' AND sistema = 'internet_banking' ORDER BY data_hora DESC;

Ferramentas úteis para coleta

  • Splunk / ELK Stack: Análise centralizada de logs
  • Wireshark: Monitoramento de tráfego de rede
  • Nessus / Qualys: Varredura de vulnerabilidades
  • Python + Pandas: Análise de grandes volumes de dados
🧠 Lembre-se: A evidência precisa ser suficiente, relevante, confiável e tempestiva.

📋 Módulo 5: Recomendações e Planos de Melhoria

Como elaborar uma boa recomendação?

Uma recomendação deve ser:

  • Específica: O que exatamente deve ser feito?
  • Factível: Dentro da realidade da organização.
  • Vinculada a um risco: Mostrar o "porquê".
  • Priorizada: O que fazer primeiro.

Estrutura de uma recomendação

📌 Exemplo de recomendação:
Risco identificado: Credenciais de ex-funcionários ativas no sistema.
Recomendação: Implementar processo automatizado de revogação de acessos integrado ao RH, com prazo máximo de 24h após desligamento, e realizar revisão trimestral de usuários ativos.
Controle esperado: Procedimento de desligamento (offboarding) documentado e auditável.

Tipos de controles para recomendar

  • Preventivos: Impedem que o erro ocorra (ex: MFA, revogação automática).
  • Detectivos: Identificam quando o erro ocorre (ex: alertas de acesso de ex-funcionários).
  • Corretivos: Minimizam o impacto após o erro (ex: plano de resposta a incidentes).

Exemplos de recomendações para riscos comuns

RiscoRecomendaçãoControle
Acesso não autorizado a contasImplementar MFA obrigatório para todos os acessos externos e revisar regras de firewallAutenticação multifator + segmentação de rede
Credenciais de ex-funcionários ativasCriar fluxo de revogação automática via integração RH-AD/LDAPProvisionamento/desprovisionamento automatizado
Falha na geração de logsConfigurar alerta de falha no serviço de logs e implementar redundância (forwarding para SIEM)Monitoramento de integridade de logs + SIEM
🚀 Dica final: Sempre que possível, priorize recomendações que automatizem controles manuais e que gerem evidências para futuras auditorias.
